# Checklists（AI自检清单）

---

## A. 检索检查（写代码前）
- [ ] 我搜索了哪些关键词？
- [ ] 我找到了哪些旧实现（>=3）？
- [ ] 我选择复用/修改哪个？理由是什么？
- [ ] 我如何确保不会重复实现？

---

## B. 避免重复实现（提交前）
- [ ] 是否出现同业务双接口/双入口？
- [ ] 是否新增了重复 Service/Util？
- [ ] 是否只是“为了更优雅”而新写一套？
- [ ] 是否明确迁移/删除旧实现？

---

## C. 架构一致性
- [ ] 新增文件是否符合模块边界？
- [ ] 命名是否符合项目规范？
- [ ] 是否有越层/跨域调用？
- [ ] 是否破坏已有接口契约？

---

## D. 质量与可维护性
- [ ] 无语法错误/类型错误
- [ ] 导入完整、变量未使用已清理
- [ ] 字段一致（DTO/Schema/接口返回值对齐）
- [ ] 日志/错误处理符合项目规范

---

## E. 留痕与交付
- [ ] CHANGELOG_AI.md 已追加记录
- [ ] 如涉及关键逻辑，PROJECT_MEMORY/NOTE 已更新
- [ ] 回复中包含：修改摘要 + 风险点 + 验证方式 + Git提交建议
